## What worked

- **Per-US cost made real, automatic, and mandatory** (`us-72`) — the headline. Modeling **the commit as the unit of attribution** (flush the main-thread watermark delta to the US the commit delivers) is simple, idempotent, and needs *no* pre-work discipline — it just rides the already-mandatory `git commit`. Proven live: the hook auto-attributed cost to the active US the moment it fired.
- **Status rollup + auto-close** (`us-65`) worked end-to-end: closing the last US rolled the feature, then the epic, to `done` and triggered the retro write — no manual step.
- **Dogfooding the gate held.** Every feature shipped as its own branch → PR → 4 green CI checks → squash-merge under branch-protection, with **zero AI attribution** in history. The process caught a real defect (see below) before merge.
- **Verifying the hook contract before building paid off** — confirming PreCompact is decision-only (it can't edit the summary) *first* turned a would-be silent no-op into the working snapshot-then-reinject design (`us-68`).

## What didn't / friction

- **Cost lumped onto one US for most of the session** — the original design only moved the active US at session boundaries, so ~$90 piled onto `us-69` while sibling US showed $0. That gap is exactly what `us-72` closed; the backfill had to redistribute by commit-diff effort because the historical main cost can't be time-split after the fact.
- **mypy was green locally but red in CI** (PR #10) — pre-commit ran pytest, not `mypy backend`, so a variable reused as two types only failed in CI. Cost a round-trip. **Lesson: run `uv run mypy backend` locally before pushing.**
- **The dogfood board JSON goes dirty on every commit** — the `tracking-attribute` hook re-saves it *after* each commit, forcing a stash → checkout → pop dance on every merge. Candidate: gitignore the live board, or accept deliberate syncs (revisited in epic-8).
- **Squash-merges leave local branches behind** — git doesn't mark a squashed branch as merged, so they piled up; `--delete-branch` only removed the remote. Surfaced as epic-8 `feat-28`.

## Decisions to revisit

- **Two attribution semantics:** main-thread cost uses a watermark **delta → current US**, subagents use a **time-sweep** over the uslog. Revisit unifying main on the time-sweep (more accurate, reconstructable) vs the added complexity.
- **Cache-read dominates the bill** (≈54% this session) — the large loaded tool/MCP prefix re-read on every call, not the visible context. Carried into epic-8 `feat-29` (transparency).
- **Expensive-but-atomic `us-72`** ($53.35) is fine to keep atomic — its cost is plumbing depth, not a missing decomposition.
