## [Unreleased]

### Changed
- **Usability refonte — cross-OS, local-first, token-efficient** (epic-10) — a sweep that makes the plugin install and run the same on **Windows / macOS / Linux** with **zero cloud CLIs**.
  - **One-command, cross-OS install + hooks.** `python scripts/bootstrap.py` replaces the manual "hand-merge `templates/.claude/settings.json`" + `bash scripts/bootstrap-smoke-test.sh` steps: it detects a working Python (≥3.9), **bakes that interpreter's absolute path** into the enforcement hooks (so they run with no PATH `python`), JSON-merges `.claude/settings.json` per event without clobbering user hooks (de-duped → re-run is a no-op), copies the enforcement hooks, runs the migrate de-dupe, and finishes with the new cross-platform `python scripts/bootstrap_smoke_test.py` (supersedes the `.sh`). See [ADR 0003](docs/adr/0003-cross-platform-tracking-and-auto-registered-hooks.md) + [ADR 0004](docs/adr/0004-cross-os-python-hook-launcher.md).
  - **No-terraform / no-gh branch-protection.** `python scripts/set_branch_protection.py --repo OWNER/REPO` (pure-stdlib `urllib`) arms the only true merge gate: it PUTs the rule via the GitHub REST API when `GITHUB_TOKEN` + a repo slug are present, else prints the exact Settings → Branches web-UI steps + the equivalent REST payload. `terraform`/`gh`/`az` drop from required to optional.
  - **Flat per-module `/fenrir:init`.** `/fenrir:init` now scaffolds **self-contained per-module projects** under `src/<module>/` (own `pyproject.toml` + `uv.lock` + `.venv`, no uv-workspace, no root lock) — see [ADR 0005](docs/adr/0005-init-module-layout.md). The `templates/ci/required-checks.yml` required-checks template became **per-module** for flat-init repos: CI iterates the module dirs (`cd src/<module> && uv sync && uv run pytest`, coverage per-module) instead of a single `uv sync --all-packages` / `--cov=src` pass at the root. `/fenrir:init` and `/fenrir:auto` arm + verify the gate via `python scripts/bootstrap.py` (in-session hooks) + `python scripts/set_branch_protection.py --repo OWNER/REPO` (branch-protection) + `python scripts/bootstrap_smoke_test.py` (verify) — **no `terraform`/`gh` required** on the canonical path.
  - **Cloud is an optional layer (local-first by default).** The plugin ships an **empty `.mcp.json`**, so the `azure`/`langfuse` MCP servers do **not** auto-start — nothing needs `az`/`terraform`/`gh`/`kubectl` to install or run. Opt in by copying a server block from `.mcp.json.example` into `.mcp.json` + setting its env vars + restarting. Shipping a PR needs only `git` (`git push` + the Compare URL). Plugin keyword `local-first`.
  - **Token-efficient delivery — `light` is the default route.** `/fenrir:deliver` defaults to **light** (inline edit or one specialist subagent); the full multi-agent pipeline is **opt-in via `--full`** or auto-triggered only for risky/large diffs (auth/iac/migrations/security path hits, or over the file/LOC size threshold). The mandatory qa-tester + red-team validation gate still runs on **every** route.
  - **Docs reconciled** across README, GETTING-STARTED, PUBLISHING, `repo-bootstrap/SKILL.md`: the bash smoke-test + manual settings merge + `terraform apply` arming are replaced with the `bootstrap.py` / `set_branch_protection.py` flow; local-dev switches from `ln -s` to `claude --plugin-dir <abs-path>` with a Windows note; a "Local-first by default" section lands in the README.

> **Epic 7 — Board automation, governance & process.** Minor bump per the new cadence: an epic finished. ### Changed
- **`release` skill — versioning cadence rule** — the SemVer bump now follows **delivery**, not a raw commit-type tally: a **feature PR merged → patch +1**; an **epic finished** (its last US reaches `done`) **→ minor +1, patch → 0**; a **breaking change → major**. A minor line accumulates one patch per feature PR and graduates to the next minor when its epic completes (the epic's retro is that minor's "what shipped"). The skill consults the board (an epic with a `→ done` transition after the last tag's date = minor) and commits (breaking detection); SKILL + VERIFY updated.

### Added
- **Subject-focused compaction** (15 hooks) — context compaction now serves the development in progress instead of producing a flat global recap. A PreCompact hook can't edit the summary text (it's decision-only), so the design is **snapshot-before, re-inject-after**: the new `precompact-focus` hook (PreCompact, on auto + manual `/compact`) captures the current dev subject — the active US (goal/acceptance), its feature+epic, and live git context (branch, recent commits, in-flight files) — to `.claude/tracking/compact-focus.md` via the engine's new `track_session.py focus` command, and surfaces it (`systemMessage`); its twin `session-context.py` (SessionStart) **re-injects that focus when `source == "compact"`**, re-grounding the compacted session on exactly what you were building (other sessions are unaffected). The ~250–300K trigger itself is Claude Code's auto-compact threshold; this makes the result subject-scoped whenever it fires. Fail-open; ships via `repo-bootstrap`.
- **Epic-close retrospective — auto-written to delivery memory** — when the last US of an epic reaches `done`, `BoardStore.set_status` auto-writes a retrospective to `docs/delivery-memory/retros/<epic>-<slug>.md` (resolved from `CLAUDE_PROJECT_DIR`/`FENRIR_RETRO_DIR`; disabled on the web server outside a repo). The doc carries the **facts** (what shipped per feature/US, the real cost rollup *Epic = Σ Features = Σ US* with cache, an opened→closed span, a status timeline) and **seeded qualitative sections** — *What worked / What didn't / Decisions to revisit* — primed from real signals (audit umbrella + expensive US, status rework) so it's never blank. It **does not clobber** a refined doc (human notes survive a reopen→reclose); `cli retro --epic <id> [--show|--force]` regenerates on demand. The `delivery-tracker` agent's doctrine gains "close the loop — refine the retro". Stored as revisitable delivery memory, **not** `MEMORY.md`.
- **Commit-time per-US cost attribution — automatic + mandatory** (14 hooks) — closes the gap where a whole session's main-thread cost lumped onto ONE User Story (the active US never moved mid-session). New `tracking-attribute` hook (PostToolUse on `git commit`) + engine command `track_session.py attribute-commit`: on **every commit**, the cost accrued **since the last commit** is flushed to the US that commit delivers — the commit is the unit of attribution ("work since the previous commit → this commit's US"). The US is resolved from the `us-N` in the commit message (the `delivery-trace` gate already makes commits reference one), else the active US, else a catch-all; it is validated against the board (a phantom US is never charged). Subagent runs stay time-swept to the US active when each ran; `tracking-finalize` (SessionEnd) is now just the backstop for any tail. Idempotent (watermark + per-run keys), fail-open (never bricks a commit), pure stdlib. Ships to consumer repos via `repo-bootstrap` (copies the hook + merges the `Bash` PostToolUse entry). **Now every US carries its real per-US cost with zero manual steps.**
- **Dashboard: US cost on the kanban card** — the story card's "US cost" now reads the authoritative `/api/board/costs` rollup (own work_log **+ its tasks**, with cache) so it matches the cost table and modal exactly, and is **always shown (incl. $0)** so a missing cost reads as a gap to fix, not a hidden value.
- **`dat-architect` agent** (14 subagents) — writes OR audits a **DAT** (Document d'Architecture Technique): the full technical-architecture document (context, component/deployment view, data, interfaces, NFRs incl. security + cost, ops, risks/alternatives, ADR log). Write mode emits `docs/dat/<slug>.md`; audit mode scores each section present/partial/missing/stale with gap→fix + a `DAT VERDICT`. Carves vs `architect` (single-decision ADRs) — the DAT references decisions, doesn't replace them.
- **Dashboard: kanban + cost-trace UX & US lifecycle** —
  - **Granularity selector** — view the kanban at **Epic / Feature / User Story** level (epic/feature cards show their cost rollup + child count; drag stays at US level).
  - **Active-only epic filter** — a `done` epic drops out of the epic dropdown (only live work clutters the view).
  - **Dynamic cost table** (Cost trace view) — **group by US / Feature / Epic** with **toggleable columns** (In / Out / Cache W / Cache R / Cost), totals row; reads the existing `/api/board/costs` rollup (Epic = Σ Features = Σ US). The chronological log stays below it.
  - **US lifecycle, move-as-you-go** — column semantics defined (backlog = idea, todo = committed, in_progress = active, review = in PR, done = merged). `/fenrir:plan` now moves planned US to **todo** (not backlog); `/fenrir:deliver` moves them **in_progress → review** as built; `/fenrir:ship` references the US in the PR (for `delivery-trace`), moves them to **review** on open and **closes them (done) after merge**. `delivery-tracker` doctrine updated.
- **Plan-first delivery workflow** (6 commands) — development starts on the board, not in the editor. New `/fenrir:plan <work>` command decomposes a piece of work into one business **Feature** + **atomic US** (reusing existing items, never an umbrella; runs `cli audit` to check granularity) and creates the `feat/<feature>` branch — **no code** at that stage. `/fenrir:deliver` and `/fenrir:challenge-me` now **check for a plan and create it if missing** (strong default, advisory — the real gate stays the `delivery-trace` CI check), and `/deliver` builds the US **one at a time** (`set-us` before each) so cost lands per-US. Convention: **one Feature = one branch = one PR**, the PR delivers that Feature's US. The `delivery-tracker` doctrine gains "plan before code".
- **Fenrir dogfoods its own PR gate (couche 0) on this repo** — `branch-protection.tf` at the root (Terraform `github` provider) requiring a PR + the real CI status checks (`dashboard (lint + type + test)`, `lint + type + test hooks`, `validate manifests`, `delivery-trace`), no direct/force push, linear history; solo-tuned (0 approvals — a lone maintainer can't self-approve). A `delivery-trace` CI workflow (`.github/workflows/delivery-trace.yml`) makes every PR reference a User Story on the board — fixed to glob the **per-project** `data/boards/*.json` (the legacy single-file `board.json` path never matched). PUBLISHING.md documents arming via `terraform apply` or one `gh api` call. The local gate (`pre-commit` install + in-session `.claude` guards) was already armed; branch-protection (the only true server-side block) needs the repo public/Pro — now satisfied.

## [1.4.0] — 2026-06-28

### Added
- **Agile-granularity audit (advisory) + atomic-US doctrine** — `BoardStore.audit()` + `cli audit` + `GET /api/board/audit` (read-only, advisory — it reports, it does not block) flag a US as an *umbrella* when it holds > a dominance share (default 40%) of a **≥3-US** epic's cost → "decompose into the real atomic US". Genuinely expensive-but-atomic US (one hard migration, a deep research run) are surfaced separately as `expensive_us` ("optimize, don't necessarily decompose"), so cost alone is never mistaken for non-atomicity. Also flags orphan US + empty features. Encodes "a $400 US is a problem" (flagged the session-umbrella us-49). The `delivery-tracker` agent's authoring **doctrine** (advisory guidance, not a hook): **US = atomic (one thing, one acceptance criterion; sub-steps are Tasks), Feature = a business/dev capability grouping its US, Epic = a program; never a per-session umbrella US; `set-us` per atomic US before the work, one US per chunk** (a single fan-out doing N US' worth of work can't be split afterward).
- **Per-US cost attribution (mandatory, automatic, with rollup)** — every spend is now traced to a *specific* US, not a per-session lump. Replaces the whole-session `link` with a `reconcile` pass: subagent runs are attributed to the US that was active **when each ran** (time-sweep over a `set-us` ledger — robust even if SubagentStop never fired), and main-thread cost to the current US as a watermark delta. main and subagent are disjoint telemetry sources → no double-count; each run attributed once (idempotent). One in-process CLI pass (`cli reconcile`, ~2s for 198 runs) called by the `tracking-finalize` hook on Stop + SessionEnd, so cost updates live. New CLI: `session-cost`, `session-runs`, `reconcile`; `cli log` gains cache + `--source`; engine gains `set-us` + a per-US `finalize`. **Rollup proven exact on real data: Epic = Σ Features = Σ User Stories** (`costs()` already aggregated; now the leaves carry real cost). Proven: 198 runs split $177/$230 across two US by time, rollup $407 consistent at every level.
- **Multi-agent cost/speed/quality efficiency** ([ADR 0001](docs/adr/0001-multi-agent-cost-efficiency.md)) — reframes the cost: cache-read is 0.1× input, so caching already saves ~10× (measured: actual ~$677 vs ~$3.5k uncached = **$2.9k saved**, 98% hit-ratio). The bill is **context volume × model tier**, not the cache.
  - **Dashboard cache-efficiency view** — `telemetry.efficiency()` + `GET /api/telemetry/efficiency` + an "Cache efficiency" panel: per model the cache hit-ratio, actual vs **uncached-equivalent** cost, and savings. A high-spend / low-hit-ratio model is the real waste to target. **Savings can go NEGATIVE** when cache writes (1.25× premium) exceed reads — surfaced as the waste signal.
  - **`workflow-efficiency` skill** (45 skills) — the authoring rubric for multi-agent workflows: tier the model per stage (mechanical → Haiku/Sonnet, judgment → Opus — the #1 overspend), budget prompts (pass file refs not inlined blobs), keep prefixes cache-stable, cap fan-out, tier effort. Advisory; the cache-efficiency view measures the result. Carves vs `context-engineering` (product context) and `llm-cost-monitor` (runtime spend).

Now version-aware: Opus 4.5–4.8 = $5/$25, Opus 4.1 = $15/$75, Fable/Mythos 5 = $10/$50, Sonnet = $3/$15, Haiku 4.5 = $1/$5, Haiku 3.5 = $0.80/$4 (cache rates still derived 1.25×/2×/0.1× input, matching the published table). ### Security hardening (red-team iteration 4 — bypasses found by EXECUTING the hooks)
- **delivery-guard.py — fail-closed + real bypass coverage.** A list-typed `command` (`["git","commit","--no-verify"]`) used to throw → fail-OPEN allow; now mutating tools fail CLOSED (deny) on any unparseable input, and `command`/`file_path` are coerced to str. Added: `cat .env`/secret-file access via Bash (not just Read), gate-bypass beyond `--no-verify` (`commit -n`, `core.hooksPath`, `HUSKY=0`, `ci.skip`, `git config hooksPath`), `re.IGNORECASE`, secret-exfil by indirection (env-var/`$(…)`/file source + outbound), non-HTTP channels (`scp`/`rsync`/`aws s3`/`gsutil`/`kubectl get secret`), more token shapes (JWT/Azure-SAS/GCP/PAT), force-push incl. `+refspec`/flag-less, `NotebookEdit`. Gate-file **kill switch** (`.claude/hooks/`, `.claude/settings*.json`) is now `deny`, not `ask`. `.env.example`/`.sample` excluded.
- **prompt-guard.py + content-scanner.py** — the canonical "ignore all previous instructions" no longer slips through (the old regex required exactly one word); broadened patterns, NFKC normalization (homoglyph/width), benign-object guard, MCP web/fetch tools now scanned.
- **session-context.py** — quote-aware YAML parse (`#` inside quotes preserved, nested/empty keys skipped), `yaml.safe_load` when available; gate-exception expiry parsed as ISO `YYYY-MM-DD` with missing/unparseable treated as **expired** (was a raw string compare that dropped valid US-format waivers and kept malformed ones).
- **config-audit.py** — now also audits `settings.local.json` (the override an attacker would use); audit-write failures warn on stderr.
- **stack-interface.yaml flattened** (`iac_backend_config` nested map → top-level keys) so the SessionStart parser doesn't drop/inject empty keys; added `func_deploy_cmd`/`vm_systemd_apply_cmd`; softened "enforces" → best-effort (real block = CI denylist).
